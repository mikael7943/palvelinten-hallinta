Sudo without password
* Ansible tarvitsee sudo-oikeudet orjakoneelle. Pitää siis saada sudokomennot toiminaan ilman salasanaa.
* Aluksi luodaan uusi käyttäjä ja ryhmä.
* Kannattaa avata uuteen ikkunaan sudo -i komento. Sudon voi korjata jos se menee rikki.
* Visudolla luodaan uusi sudoers sääntö, jonka avulla sudoa voi käyttää ilman salasanaa. laitetaan koodi %sudoless ALL = (ALL) NOPASSWD: ALL. Tämän jälkeen sudon pitäisi toimia ilman salasanaa.

Sandwitch
* Piirroksessa viitataan sudo komentoon, jonka avulla voi ajaa mitä tahansa komentoja.
* Aluksi toinen hahmo ei halua tehdä leipää, mutta sudon jälkeen hän on paljon halukkaampi leivän tekemiseen.

Passwordless sudo with ansible
* Luodaan ansible-playbook, jonka avulla päästään ajamaan sudo komentoja ilman salasanaa.
* Luodaan uusi rooli ja hakemistot.
* yml-tiedostoon kirjoitetaan käyttäjä ja sen oikeudet jne.


 Ansible-doc
 *ansible-doc copy kopioi tiedoston tai hakemiston etäkoneelta tai paikallislta koneelta etäkoneelle.

 optiot:
 
 content: Asettaa tiedoston sisällön suoraan määritettyyn arvoon
 dest: etäinen absoluuttinen polku minne tiedosto kopioidaan
 group: Ryhmän nimi, joka omistaa tiedostojärjestelmäobjektin
 mode: luvat osoitettuun tiedostoon tai hakemistoon
 owner: tiedostojärjestelmän omistaja
 src: paikallinen polku tiedostoon, johon etäpalvelin kopioidaan.

 Esimerkki:
 - name: Copy file with owner and permissions
   ansible.builtin.copy:
     src: /srv/myfiles/foo.conf
     dest: /etc/foo.conf
     owner: foo
     group: foo
     mode: u=rw, g=r, o=r
   
   
*Ansible-doc apt
 name: lista pakettien nimistä tai niiden tarkoista versioista
 state: ilmaisee paketin halutun tilan. 'latest' takaa että uusin versio on asennettu.
 update_cache: Ajaa vastaavan komennon kuin 'apt-et update' ennen oikean komennon ajamista
 
Esimerkki:
- name: Install apache httpd
  ansible.builtin.apt:
    name: apache2
    state: present

optiot:

path: polku manaregoitavaan tiedostoon
recurse: aseta rekusrsiivisesti määritelty tiedostoattribuutti hakemiston sisälle
src: linkitettävän tiedoston polku

*Ansible-doc user
Manageroi käyttäjiä and niiden attribuutteja

optiot:
create_home: Ellei ole asetettu 'false' luodaan käyttäjälle kotihakemisto
comment: asettaa valinnallisesti käyttäjätilin kuvauksen
groups: luettelo täydenävistä ryhmistä, joihin käyttäjä myös kuuluu
name: luotavan, poistettavan tai modifoitavan käyttäjän nimi
shell: asettaa valinnaisesti käyttäjän komentotulkin
state: onko tilin oltava olemassa vai ei, ja toimenpiteisiin ryhtyminen, jos tila poikkeaa hyväksytystä
system: käyttäjää luodessa 'state=present' laittaessa 'true' tekee käyttäjästä järjestelmätilin

Esimerkki:
- name: Add the user 'johnd' with a specific uid and a primary group of 'admin'
  ansible.builtin.iser:
    name: John Doe
    comment: John Doe
    uid: 1040
    group: admin
  

*Ansible-doc authorized_key
Lisää tai poistaa SSH avaimen tietyiltä käyttäjiltä

optiot:
key: SSH julkinen avain merkkijonona tai url-osoitteena
user: Käyttäjä, jonka authorozed_keys tiedostoa muokataan.


Esimerkki:
- name: Set authorized key taken from file
  ansible.posix.authorized_key
    user: charlie
    state: present
    key: "{{ lookup('file', '/hom/charlie/.ssh/id_rsa.pub') }}"
